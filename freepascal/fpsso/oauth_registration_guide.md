# คู่มือการขอ API Key และ Secret Key (OAuth 2.0)

การเชื่อมต่อระบบ Single Sign-On (SSO) จำเป็นต้องใช้ `Client ID` และ `Client Secret` จากผู้ให้บริการแต่ละราย เพื่อใช้ยืนยันตัวตนว่าระบบของเราได้รับอนุญาตให้เชื่อมต่อได้อย่างถูกต้อง ด้านล่างนี้คือขั้นตอนและช่องทางการขอ Key ของแต่ละระบบครับ

---

## 1. Google OAuth 2.0 (Google Sign-in)
การขอ Key ของ Google สามารถทำได้ด้วยตัวเองผ่าน Google Cloud Console ฟรีและรวดเร็วที่สุด

**ขั้นตอนการดำเนินการ:**
1. ไปที่เว็บ **[Google Cloud Console](https://console.cloud.google.com/)** และล็อกอินด้วยบัญชี Google
2. สร้างโปรเจกต์ใหม่: 
   - คลิกที่ชื่อโปรเจกต์ด้านบนซ้ายมือ เลือก **New Project** ตั้งชื่อ (เช่น `FP-SSO-Portal`) แล้วกด Create
3. ตั้งค่าหน้าจอขออนุญาต (Consent Screen):
   - ไปที่เมนูซ้ายมือ เลือก **APIs & Services** > **OAuth consent screen**
   - เลือก **External** (หากต้องการให้ใครก็ได้ที่มี Gmail ล็อกอินได้) แล้วกด Create
   - กรอกข้อมูลที่จำเป็น เช่น ชื่อแอป (App name), อีเมลติดต่อ (User support email) แล้วกด Save and Continue จนจบ
4. สร้าง Credentials:
   - ไปที่เมนู **Credentials** ด้านซ้ายมือ
   - คลิก **+ CREATE CREDENTIALS** ด้านบน เลือก **OAuth client ID**
   - **Application type:** เลือก `Web application`
   - **Name:** ตั้งชื่อตามต้องการ (เช่น `SSO Web Client`)
   - **Authorized redirect URIs:** ให้กรอก URL ที่ระบบจะเด้งกลับมาหา (ตามที่ตั้งไว้ในตัวแปร `.env`)
   - กด **Create**
5. **เสร็จสิ้น:** ระบบจะแสดงหน้าต่าง Popup ที่มี **Client ID** และ **Client Secret** ให้นำค่านี้ไปใส่ในไฟล์ `.env` ของเราครับ

---

## 2. ThaID (แอปพลิเคชัน ไทยดี)
ThaID เป็นระบบของ กรมการปกครอง (DOPA) กระทรวงมหาดไทย การขอเชื่อมต่อจะต้องทำในนามหน่วยงานของรัฐหรือนิติบุคคลที่ได้รับอนุญาต

**ขั้นตอนการดำเนินการ:**
1. **เตรียมเอกสาร:** หน่วยงานของคุณจะต้องทำ **หนังสือราชการ (หรือหนังสือขอความอนุเคราะห์)** เรียน ผู้อำนวยการสำนักบริหารการทะเบียน กรมการปกครอง
2. **ระบุรายละเอียดในหนังสือ:**
   - วัตถุประสงค์ในการขอเชื่อมต่อระบบ ThaID (เช่น เพื่อยืนยันตัวตนผู้ใช้งานระบบเครือข่ายอินเทอร์เน็ตภายในโรงพยาบาล)
   - ข้อมูลของเซิร์ฟเวอร์ เช่น **Public IP Address** ของระบบ SSO
   - **Redirect URI** ที่จะให้ระบบ ThaID ส่งข้อมูลกลับมา
   - ชื่อและเบอร์โทรศัพท์ผู้ประสานงานด้านเทคนิค
3. **การพิจารณาและรับ Key:**
   - เมื่อกรมการปกครองอนุมัติ ทางทีมเทคนิคของ DOPA จะส่งข้อมูล **Client ID** (App ID), **Client Secret** และบางกรณีอาจมี **API Key** มาให้ทางอีเมลของผู้ประสานงาน
4. นำค่าที่ได้มาใส่ใน `.env` ในหมวดของ `THAID_CLIENT_ID` และ `THAID_SECRET_ID`

---

## 3. ProviderID / MOPH ID
ระบบ ProviderID (สำหรับบุคลากรทางการแพทย์) และ MOPH ID (สำหรับประชาชน) ดูแลโดย ศูนย์เทคโนโลยีสารสนเทศและการสื่อสาร (ICT) สำนักงานปลัดกระทรวงสาธารณสุข

**ขั้นตอนการดำเนินการ:**
1. **การลงทะเบียนผู้ดูแลระบบ:** ผู้ดูแลระบบจะต้องมีบัญชี MOPH ID / ProviderID อยู่แล้ว และได้รับสิทธิ์ให้เข้าถึงหน้าจัดการ Developer Portal ของกระทรวงสาธารณสุข
2. **สร้าง Application ใหม่:**
   - เข้าสู่ระบบจัดการ API ของกระทรวงสาธารณสุข (URL จะขึ้นอยู่กับประกาศของศูนย์ ICT กระทรวงฯ ณ ปัจจุบัน)
   - เลือกเมนูสำหรับนักพัฒนา (Developer) และสร้าง **New Application** หรือ **New Client**
3. **ตั้งค่าระบบ:**
   - ระบุชื่อระบบของคุณ (เช่น `ระบบยืนยันตัวตนเครือข่าย (Wi-Fi) รพ. ...`)
   - ระบุ **Redirect URI** ที่จะใช้รับค่ากลับมา
4. **รับ Key:**
   - ระบบจะสร้าง **Client ID** และ **Client Secret** ให้ทันทีบนหน้าเว็บ
5. นำค่าเหล่านี้ไปใส่ในไฟล์ `.env` ในส่วนของ `PROVIDER_ID_CLIENT_ID` และ `PROVIDER_ID_SECRET_KEY`

> **หมายเหตุ:** สำหรับ ThaID และ ProviderID หากมีการเปลี่ยนแปลง URL ของเว็บไซต์ (Redirect URI) คุณจะต้องติดต่อแจ้งผู้ดูแลระบบของทางกรมการปกครอง หรือศูนย์ ICT ทุกครั้งเพื่อเพิ่ม URL ใหม่ลงในระบบของเขา มิฉะนั้นผู้ใช้จะเจอข้อผิดพลาด "Invalid Redirect URI" ครับ
